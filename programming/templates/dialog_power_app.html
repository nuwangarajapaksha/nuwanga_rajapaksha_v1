Dialog Power App – Alarm and Notification Management System

My Role: Full-Stack Software Engineer
Technology Stack: Java, Spring Boot, MySQL, Hibernate, JPA, Microservices, ReactJS, Redux, JavaScript, HTML, CSS, Material-UI, Bootstrap, Flutter, Android, JSON, Postman, Swagger, JUnit, Git, GitLab, Jira, Agile, AWS, Docker

Project Overview

Dialog Power App is a mission-critical mobile and web-based application developed by XdotO Concepts Pte Limited in collaboration with Dialog Axiata PLC, Sri Lanka’s leading telecom provider. This system was designed to modernize the alarm and notification management process across Dialog’s extensive infrastructure—including signal towers, distribution centers, maintenance sites and etc.

The application enables field officers, such as site managers and regional managers, to receive, track, and respond to various real-time alarms and notifications—including theft alerts, low battery warnings, and maintenance requests. It improves operational responsiveness, facilitates task coordination, and generates actionable insights for management.

Architecture

The system architecture consists of three main components:

  • Mobile App: Deployed to officers in the field for real-time alarm monitoring, acknowledgment, and task processing. Built with Flutter for cross-platform support and Native Android for optimized performance.
  • Admin Web Panel: Used by administrators to manage users, monitor site-level activities, generate reports, and review performance analytics. Developed with ReactJS and Redux, styled using HTML, CSS, Material-UI, and Bootstrap.
  • Backend Platform: Built using a microservices architecture, the backend handles alarm processing, user and role management, workflow handling, and analytics. Implemented with Java and Spring Boot, using MySQL as the database, with JPA and Hibernate for ORM, and JSON for data communication between services and the frontend.

Key Features

  • Real-time alarm creation and notification handling (manual and automated via sensors and machines)
  • Alarm types include theft, intrusion, battery low, maintenance requests, and etc
  • Enabling efficient monitoring and quick response to system events
  • Role-based task routing for efficient response workflows
  • Admin dashboard with activity logs, performance tracking, and reporting
  • User authentication, authorization, and role-based access control
  • Automated daily and historical analysis reports
  • Robust error handling, diagnostics, and system monitoring
  • Cloud deployment via Docker and AWS for scalability and availability
