Morex – Process Monitoring System

My Role: Full-Stack Software Engineer
Technology Stack: PHP, MySQL, VueJS, JavaScript, HTML, CSS, Bootstrap, Grafana, Git, GitLab, Jira, Agile

Project Overview

Morex is a comprehensive Process Monitoring System developed by XdotO Concepts Pte Limited as a sub-project for Morex Industries. Designed as a web-based application, it streamlines and digitizes the production workflows and operational monitoring of industrial chemical and material processing. The system enables users to manage end-to-end production workflow sand sub-production workflows, inventory, error corrections, history tracking, alarms, reporting, real-time monitoring, and graphical visualizations through a role-based interface.

The application supports five parallel production workflows, referred to as "Reactors." Each reactor represents a complete production line involving multiple steps—such as machine status checks, material and chemical addition, and data logging—with predefined templates to simplify execution. Users can interact with each workflow in real time, monitor progress, and handle deviations or alarms as needed.

To enhance operational visibility, the system enables real-time analysis of production processes, ensuring improved efficiency, reduced downtime, and data-driven decision-making within the manufacturing environment.

Architecture

  • The application is built on a monolithic PHP backend with VueJS, JavaScript, HTML, CSS and Bootstrap for the frontend. MySQL is used as the relational database to store production data, templates, inventory logs, and more. Grafana was integrated into the system to visualize live sensor data such as machine temperature, thresholds, and other key metrics. The entire application is deployed on a Linux-based on-premise server.

Key Features

  • Role-based sign-in and access control for secure multi-user operations
  • Production section featuring five parallel workflow reactors with real-time progress indicators
  • Predefined templates to standardize production processes and improve usability
  • Inventory module to track availability and usage of materials and chemicals
  • History section with searchable logs and summary reports of completed productions
  • Real-time machine and reactor monitoring, including temperature and operational status
  • Job sheet module for sub-production documentation and tracking
  • Datalogger status monitoring with live sensor data visualized via Grafana
  • Error correction module for resolving workflow issues post-execution
  • Alarms and notifications system with escalation handling and response management
  • Reporting section with multiple report types including production, inventory, and job sheets
  • Admin panel for managing users, templates, materials, and chemicals
